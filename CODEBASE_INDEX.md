# FINLLAMA Codebase Index

## Repository Overview
This repository contains a reproducible research pipeline for financial sentiment analysis using a LoRA-fine-tuned large language model.
The implementation is structured to support gated dataset ingestion, automated teacher-label generation, model adaptation with parameter-efficient fine-tuning, inference, quantitative evaluation, and portfolio strategy analysis.

## Primary Repository Files

- `README.md`
  - Provides a structured project overview, installation guidelines, dependency requirements, and a concise summary of the end-to-end workflow.

- `requirements.txt`
  - Enumerates the Python libraries required to execute the pipeline, including core machine learning, dataset, and analysis libraries.

- `CODEBASE_INDEX.md`
  - Documents the repository hierarchy and describes the purpose of each major file and submodule.

## Data Assets

- `data/financial_news_shards/`
  - Contains unlabeled Parquet data shards produced from the gated Hugging Face financial news dataset.
  - Each shard preserves normalized text and associated timestamps for downstream processing.

- `data/financial_news_shards_labeled/`
  - Contains the same text shards annotated with sentiment labels and confidence scores generated by the FinBERT teacher model.

- `data/llm_training_data/`
  - Contains the final dataset partitions used for LLM fine-tuning and evaluation: training, validation, and test splits.

## Model Storage

- `models/Llama-3.2-1B/`
  - Stores the base Llama model components and artifacts used for baseline evaluation and adapter-based fine-tuning.

- `models/finllama_lora/`
  - Stores the trained LoRA adapter weights and configuration for the FinLLaMA model.

- `models/finbert-tone/`
  - Stores the FinBERT model artifacts used for pseudo-label generation in the data preparation pipeline.

## Results and Visualizations

- `results/predictions/`
  - Houses CSV files containing model inference outputs for the baseline and tuned models.

- `results/figures/`
  - Houses evaluation plots and backtest visualizations generated by the analysis modules.

## Source Code Structure

### `src/data_prep/`
This module implements data ingestion, labeling, and supervised dataset construction.

- `financial_multisource_loader.py`
  - Streams the Hugging Face financial news dataset in authenticated mode.
  - Applies text normalization, duplicate elimination, and shard-level persistence to Parquet format.

- `label_finbert_on_parquet_shards.py`
  - Loads unlabeled text shards and infers sentiment labels using the FinBERT classification model.
  - Emits labeled Parquet shards containing sentiment codes and per-example confidence estimates.

- `prepare_llm_data.py`
  - Aggregates labeled shards, samples data with temporal stratification, and creates the training, validation, and test partitions required for LLM adaptation.

### `src/modeling/`
This module encapsulates model training and inference workflows for both baseline and fine-tuned model variants.

- `train_model.py`
  - Implements the FinLLaMA fine-tuning workflow.
  - Loads the base Llama model, configures quantization, applies LoRA adapters, and executes parameter-efficient training using the Hugging Face Trainer API.

- `run_finllama_inference.py`
  - Attaches trained LoRA adapters to the base model and performs inference on the held-out test partition.
  - Produces a CSV artifact containing predicted sentiment labels and confidence values.

- `run_base_llama_inference.py`
  - Executes baseline inference using the unadapted base Llama model.
  - Produces a CSV artifact for benchmark comparison.

### `src/evaluation/`
This module provides systematic model performance assessment and visualization.

- `evaluate_all.py`
  - Computes classification metrics, including accuracy, weighted F1 score, and class-specific precision/recall.
  - Generates a confusion matrix and comparative performance visualization.

- `plot_roc.py`
  - Computes one-versus-rest ROC curves and AUC values for the tuned model predictions.
  - Produces graphical evaluation artifacts for diagnostic analysis.

### `src/portfolio/`
This module translates sentiment predictions into portfolio performance analysis.

- `backtest_portfolio.py`
  - Converts daily sentiment signals into long/short position recommendations.
  - Fetches market price data and simulates strategy performance.
  - Computes portfolio statistics and generates cumulative return visualizations.

### `src/setup/`
This module contains environment and authentication utilities.

- `setup_huggingface_auth.py`
  - Provides an interactive mechanism to register Hugging Face authentication credentials.
  - Supports access to gated datasets required by the data ingestion workflow.

## Recommended Execution Workflow

1. Establish Hugging Face authentication using `src/setup/setup_huggingface_auth.py`.
2. Execute `src/data_prep/financial_multisource_loader.py` to produce raw Parquet shards.
3. Execute `src/data_prep/label_finbert_on_parquet_shards.py` to generate teacher-labeled sentiment shards.
4. Execute `src/data_prep/prepare_llm_data.py` to generate the final training, validation, and test datasets.
5. Execute `src/modeling/train_model.py` to fine-tune the FinLLaMA model.
6. Execute `src/modeling/run_finllama_inference.py` and `src/modeling/run_base_llama_inference.py` to generate prediction artifacts.
7. Execute `src/evaluation/evaluate_all.py` and `src/evaluation/plot_roc.py` for model benchmarking and visualization.
8. Execute `src/portfolio/backtest_portfolio.py` to evaluate the investment strategy implications of sentiment predictions.

## Research Notes

- The pipeline employs a teacher-student paradigm in which FinBERT supplies pseudo-labels and the FinLLaMA model learns to approximate those sentiment annotations.
- Parameter-efficient fine-tuning is achieved through LoRA adapters and quantization strategies to reduce resource requirements.
- The codebase is organized to separate data acquisition, label generation, training, evaluation, and portfolio simulation into distinct reproducible stages.
